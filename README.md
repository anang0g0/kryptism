さて去年からの成果物をここに説明しておきます。置換群の共役元を使った暗号は連立方程式で解ける場合と、解けない場合があるということで確実な解読法とは言えないことが実験の結果わかりました。具体的には、共役元を未知の行列C、公開鍵をWとして定義したとき、その変数行列Cを右からかけてやり、WC=CYとして連立方程式を解くという問題に還元させるというものでした。しかし、一般の行列と異なり、置換行列は要素が１しかなく非常に自由度の少ない行列であるため、必要な数の方程式が得られません。このため、各変数がリンクされている場合は行の各要素に１つづつ１を代入して総探索をかければ、行列の次元ｎ回だけ試せば決定できますが、それぞれの変数が線形独立な場合は組み合わせ論的に難しくなります。そこで攻撃法と呼べるような確実性がなくなってしまいます。もしこの方法が安全でないとした場合に、同じ攻撃関数を復号に使うことで第二の成果物の復号関数に使えることがわかります。第二の成果物として以下の暗号系を提示します。</br>
</br>
置換群を用いた公開鍵暗号</br>
秘密鍵X,Y,a（n次の置換群の元 X、Y  (XとYは非可換)）</br>
</br>
アリスの公開鍵</br>
W1=XYX^-1</br>
W2=X^aYX^-a</br>
</br>
ボブの暗号化</br>
C1=W1^ｰrW2^r</br>
</br>
復号</br>
W1^-rW2^r</br>
=XY^-rX^(a-1)Y^rX^-a</br>
->C'=X^-1W1^rW2^rX^a=Y^-rX^(a-1)Y^r</br>
->Y^rC'=X^(a-1)Y^r</br>
ここで、X,X^(a-1)はアリスにだけわかる秘密の置換</br>
故にW1^rW2^rから、連立方程式を解くことで未知の置換行列Y^rがわかり、Y^rとYからｒが決定できる。もしくはケーリーハミルトンでYの乗数を決定する。
</br>
これもまだ確定はしていないのですが、もしｒを決定する方法があれば共役元探索問題の代わりに使える方法だと思います。この暗号は、秘密鍵を知らなければ、式変形ができず、また復号に必要なｒを決定できないところにあります。置換群の離散対数問題自体は、サイクルの最小公倍数を決定することで計算できるのですが、この方式の場合、置換群X,Yのどちらも公開されていないので、そもそもどのような秘密鍵を使って公開鍵が作られたかを決めることができません。しかし、全てを知っているアリスにとっては、未知の乗数をYから決定できるという仕掛けです。

秘密鍵暗号はなくならない？ツイッタ上でインフラエンジニアの偉い人と議論する機会があったのですが、秘密鍵暗号は方式として独立しているので公開鍵暗号によって買えることができないという話をしました。最初、そんなことはないだ漏斗思っていたのですが、公開鍵だと毎回鍵を更新する必要があるのでトラフィックが増大する。秘密鍵も毎回更新すれば増大するけど、その必要があるのかどうか。毎回鍵交換をしなくていいのは、同じ鍵を使いつづけることが前提だから。公開鍵の場合は毎回変えないとランダム性がなくなるので暗号化してないのと同じ事になる。あれでもそれって秘密鍵暗号でも同じでは？ECBモードだと同じですよね？CTRとかどこかでランダム性を確保するわけだし。共有する乱数列があればいいけど。どっちも変わんないんじゃないのかなー？秘密鍵暗号で毎回鍵を更新する場合が公開鍵暗号だけを使う場合と同じなのか？ちなみに使い捨て公開鍵は情報理論的に安全です。
